# Visió del producte

## Context actual
- Tenim un esdeveniment bisetmanal anomenat "Olimpiades".
- Cada dia es un esport diferent amb equips que van de 1 persona a infinites per equip segons els esports.
- Un cop a l'any es decideixen quins esdeveniments es faran i es pot coneixer els esports abans del dia.
- Actualment l’assistència es comunica principalment per WhatsApp.
- Després dels esdeveniments, els resultats es passen manualment a un Excel.
- L’Excel s’utilitza per sumar i restar punts i mantenir una classificació general.
- El producte va per temporades cada una d'elles amb X events que es defineixen dos cops a l'any aproximadament

## Problema actual
- El procés és manual.
- La informació està dispersa entre WhatsApp i Excel.
- Fer el recompte de punts porta temps.
- És fàcil cometre errors en la classificació.
- No hi ha una eina centralitzada per consultar historial, assistència i resultats.
- Ha d'haver un administrador pero un segon rol amb menys permisos que pugui fer funcionalitats com afegir puntuacions o crear equips.

## Objectiu
Crear una web app que permeti gestionar participants, assistència, equips, resultats i classificació. Aquesta web app ha de ser adaptable responsive per mobils i tablets.

## Objectius de l’MVP
- Donar d’alta participants.
- Crear un esdeveniment.
- Registrar assistència.
- Assignar participants a equips.
- Registrar posicions o resultats.
- Calcular punts automàticament.
- Mostrar classificació general.

## Restriccions i prioritats
- Plataforma principal: Google Cloud.
- Prioritat a cost nul o molt baix en MVP.
- Solució simple i mantenible.
- Evitar sobreenginyeria.
- Integració amb WhatsApp es considera opcional o futura.
